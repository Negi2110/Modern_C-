```cpp
// ============================================================
// STATIC MEMBER VARIABLES AND FUNCTIONS IN CLASSES - INTERVIEW NOTES
// ============================================================

#include <iostream>

// ============================================================
// SECTION 1: YOUR COMMENT - "STATIC VARIABLE EXISTS OUTSIDE THE CLASS"
// ============================================================

// YOUR COMMENT: "static variable in a class exists outside of the class,
//                the scope is within the class however"
//
// THIS IS THE KEY INSIGHT - let me break it down precisely:
//
// SCOPE  = where in code can you ACCESS/NAME the variable?
//          -> API::MAJOR_VERSION (must use API:: to name it)
//          -> only "findable" through the API class scope
//          -> you cannot just write MAJOR_VERSION (compiler error)
//
// STORAGE = where does the variable PHYSICALLY LIVE in memory?
//           -> DATA SEGMENT (same as global variables)
//           -> NOT inside any object/instance
//           -> NOT on the stack, NOT on the heap
//           -> exists even if ZERO API objects have been created
//
// SO:
// SCOPE    = CLASS (accessed via API::)    <- scoped TO the class
// STORAGE  = DATA SEGMENT (global-like)   <- exists OUTSIDE any object
//
// THIS EXPLAINS:
// int API::MAJOR_VERSION = 7;  // defined OUTSIDE the class body!
// -> the class just DECLARES it (says it will exist)
// -> you must DEFINE it outside (actually create the storage)
// -> exactly like declaring a function in .hpp, defining in .cpp

// ============================================================
// SECTION 2: THE THREE FILES - HEADER, IMPLEMENTATION, MAIN
// ============================================================

// ---- api.hpp (INTERFACE - what users see) ----
//
// struct API {
// public:
//     API();
//     ~API();
//
//     int m_local;           // INSTANCE member - each object gets its OWN
//
//     static int MAJOR_VERSION;   // DECLARATION only (no storage yet)
//     static int MINOR_VERSION;   // tells compiler: "this will exist somewhere"
//
//     static int GetMajorVersion();  // static function declaration
//     // note: 'static' keyword written ONLY in declaration (.hpp)
//     // NOT repeated in definition (.cpp) - same rule as virtual!
// };

// ---- api.cpp (IMPLEMENTATION) ----
//
// API::API() {}
// API::~API() {}
//
// int API::GetMajorVersion()   // NO 'static' keyword here!
// {                            // (just like 'virtual' not repeated in .cpp)
//     return MAJOR_VERSION;    // can access static member directly
//     // return m_local;       // ERROR - m_local belongs to an instance
//                              // no 'this' pointer in static function!
// }
//
// int API::MAJOR_VERSION = 7;  // DEFINITION - this creates the actual storage
//                               // in the data segment
//                               // ONE copy for ALL API objects
// YOUR COMMENT: "don't write static here because it is already
//                declared as static in the class" - EXACTLY RIGHT
// Same rule as virtual:
// -> 'static' in .hpp declaration = tell compiler it's static
// -> 'static' in .cpp definition  = DIFFERENT MEANING (file scope)
//                                    = would be a DIFFERENT variable!
//                                    = WRONG and confusing -> never do it

// ---- main.cpp ----
//
// API::MAJOR_VERSION   // access WITHOUT any object (scope resolution)
// API::GetMajorVersion() // call WITHOUT any object
//
// instance.MAJOR_VERSION // also valid but misleading (looks instance-specific)
//                        // prefer API:: to make it clear it's shared

// ============================================================
// SECTION 3: static MEMBER VARIABLE - SHARED ACROSS ALL INSTANCES
// ============================================================

// struct API {
//     int m_local;              // each instance has ITS OWN m_local
//     static int MAJOR_VERSION; // ALL instances share ONE MAJOR_VERSION
// };
//
// MEMORY VISUAL:
//
// DATA SEGMENT:
// [MAJOR_VERSION = 7]   <- ONE copy for EVERYONE
//
// STACK (if you create instances):
// API a1: [m_local = ?]  <- a1's own m_local
// API a2: [m_local = ?]  <- a2's own m_local (different from a1!)
//
// a1.m_local = 5;    // affects ONLY a1
// a2.m_local = 10;   // affects ONLY a2
// a1.m_local != a2.m_local   // each instance has own copy
//
// API::MAJOR_VERSION = 7;  // ONE value, ALL objects see the same
// a1.MAJOR_VERSION = 8;    // changes it for EVERYONE (a2 also sees 8!)
// a2.MAJOR_VERSION         // = 8 (same storage!)
//
// THIS is what "static member = belongs to the CLASS not instances" means

// ============================================================
// SECTION 4: static MEMBER FUNCTION - NO 'this' POINTER
// ============================================================

// static int GetMajorVersion()
// {
//     std::cout << this << '\n';  // COMPILER ERROR!
//     // "this" not available in static function
//     // 'this' = pointer to current INSTANCE
//     // static function has NO instance -> no 'this'
//
//     return MAJOR_VERSION;  // OK - static member, exists without instance
//
//     return m_local;        // ERROR - m_local belongs to an instance
//                            // which instance? there's no 'this'!
//                            // compiler refuses
// }
//
// RULE: static function can ONLY access:
// -> other static member variables
// -> other static member functions
// -> local variables inside itself
// -> global variables
// CANNOT access:
// -> non-static member variables (need 'this')
// -> non-static member functions (need 'this')

// ============================================================
// SECTION 5: const static - YOUR COMMENT
// ============================================================

// YOUR COMMENT: "better to put const on static"
//
// static int MAJOR_VERSION = 7;          // can be changed at runtime
// const static int MAJOR_VERSION = 7;    // LOCKED, cannot change
// static const int MAJOR_VERSION = 7;    // same thing (order doesn't matter)
//
// BENEFIT of const static:
// -> MAJOR_VERSION is a version number - it should NEVER change at runtime
// -> const prevents accidental modification: API::MAJOR_VERSION = 8; -> ERROR
// -> documents intent: "this is a constant value, not a mutable counter"
//
// BONUS: const static INTEGRAL types can be initialized IN the class:
// struct API {
//     static const int MAJOR_VERSION = 7;    // OK for integral type in header!
//     static const float PI = 3.14f;         // NOT OK (non-integral)
// };
// -> no need for separate definition in .cpp for const integral static!
// -> compiler often inlines the value directly (no actual storage)
//
// C++17 inline static:
// struct API {
//     inline static int MAJOR_VERSION = 7;   // definition IN header, C++17
// };
// -> also no separate .cpp definition needed
// -> works for any type (not just integral)

// ============================================================
// SECTION 6: WHY NO INSTANCE NEEDED - THE "API" USE CASE
// ============================================================

// YOUR COMMENT: "no instance of API"
// This is a NAMESPACE-LIKE use of a class:
//
// API::MAJOR_VERSION    // access without creating any API object
// API::GetMajorVersion() // call without any API object
//
// WHEN to use static members this way:
// 1. VERSION INFORMATION: API::MAJOR, API::MINOR (your code)
// 2. UTILITY FUNCTIONS: Math::sin(), Math::cos() (no Math object needed)
// 3. FACTORY FUNCTIONS: Connection::create() (creates objects of a type)
// 4. COUNTERS: Person::getCount() (how many Person objects exist)
// 5. SINGLETON: Database::getInstance() (return the one shared instance)
//
// ALTERNATIVE: namespace (often preferred for pure utility):
// namespace API {
//     const int MAJOR_VERSION = 7;
//     int GetMajorVersion() { return MAJOR_VERSION; }
// }
// API::MAJOR_VERSION   // same syntax!
// namespace is preferred when there's no need for inheritance/polymorphism

// ============================================================
// SECTION 7: DEFINITION OUTSIDE CLASS - WHY MANDATORY (usually)
// ============================================================

// struct API {
//     static int MAJOR_VERSION;  // DECLARATION (in .hpp)
//                                // tells compiler: "this EXISTS"
//                                // but creates NO STORAGE yet
// };
//
// int API::MAJOR_VERSION = 7;    // DEFINITION (in .cpp)
//                                // creates ACTUAL STORAGE in data segment
//                                // ONE definition rule applies
//                                // if missing: LINKER ERROR "undefined reference"
//
// WHY must definition be in .cpp (not .hpp)?
// -> If definition is in .hpp and .hpp is included in 3 .cpp files:
//    -> 3 definitions of the same variable!
//    -> LINKER ERROR: "multiple definition of API::MAJOR_VERSION"
// -> .cpp is compiled ONCE -> definition appears exactly ONCE -> correct
//
// EXCEPTION: const static integral (no separate definition needed)
// struct API { static const int V = 7; };  // complete in header!

// ============================================================
// SECTION 8: INTERVIEW QUESTIONS
// ============================================================

// Q1: What is a static member variable?
//     -> A variable that belongs to the CLASS, not any specific instance.
//        ONE copy shared by ALL objects. Stored in data segment.
//        Exists even when zero instances exist.

// Q2: What does "scope within the class, storage outside" mean?
//     -> SCOPE: accessed via ClassName:: (found through class namespace)
//        STORAGE: physically lives in data segment like a global variable,
//        NOT inside any object's memory. This is exactly what your
//        comment described.

// Q3: Why must static member variables be defined outside the class?
//     -> Declaration in class (header) says it exists.
//        Definition in .cpp creates actual storage in data segment.
//        If in header, multiple .cpp inclusions = multiple definitions
//        = linker error. One .cpp file = one definition.

// Q4: Why can't static member functions access non-static members?
//     -> Static functions have no 'this' pointer.
//        Non-static members (like m_local) belong to a specific instance.
//        Without 'this', there's no way to know WHICH instance's
//        m_local to access.

// Q5: Why don't you write 'static' in the .cpp definition?
//     -> 'static' on a free function in .cpp means FILE SCOPE
//        (internal linkage) - a completely different meaning!
//        The class declaration already marks it as static.
//        Repeating it in .cpp would create a conflict.
//        Same reason 'virtual' isn't repeated in .cpp.

// Q6: What is the difference between static member variable
//     and a global variable?
//     -> Global: accessible anywhere by name, no class scope.
//        Static member: must use ClassName:: to access, scoped to class.
//        Both live in data segment. Static member provides
//        better encapsulation (private static is possible).

// Q7: When should you use const static?
//     -> When the shared class value should NEVER change at runtime.
//        API version numbers, mathematical constants, configuration values.
//        const prevents accidental modification.
//        const integral static can be initialized inside the class header.

// Q8: What is the difference between static member function
//     and a namespace function?
//     -> Static member: can be private (access control), participates
//        in inheritance, can access private static members.
//        Namespace function: no access control, simpler, preferred
//        for pure utility functions with no class state needed.

int main()
{
    // Access WITHOUT creating any API instance:
    std::cout << "Major: " << API::MAJOR_VERSION << '\n';
    // prints: Major: 7

    std::cout << "Major: " << API::GetMajorVersion() << '\n';
    // prints: Major: 7

    // Could also do (but misleading):
    // API instance;
    // instance.MAJOR_VERSION  // works but looks instance-specific
    // prefer API:: syntax always for static members

    return 0;
}
```

**The one statement that explains everything:**

```
static member variable:
SCOPE   = CLASS (accessed via API::)     ← "within the class"
STORAGE = DATA SEGMENT (like global)     ← "exists outside the class"

This is EXACTLY what your comment said!
```

**The four rules for static member functions:**

```cpp
static int GetMajorVersion() {
    return MAJOR_VERSION;  // ✅ static member - OK
    return m_local;        // ❌ instance member - no 'this'!
    std::cout << this;     // ❌ no 'this' in static function
    GetMinorVersion();     // ✅ if also static - OK
}
```

> **Don't write `static` in the `.cpp` definition** — same rule as `virtual`. In the `.hpp` declaration, `static` means "class-level shared". In a `.cpp` free function, `static` means "file-scope only" — completely different! The compiler would either reject it or interpret it wrongly.
